# 拾词 — 项目说明

## 项目概述

"拾词"：一个帮助高中生检测英语单词掌握情况的网页工具。单 HTML 文件，打开即用。

## 文件结构

```
coding/
├── CLAUDE.md              ← 本文件，项目总指引
├── 单词测试.html           ← 主程序（单个 HTML 文件，约 2200 行）
├── docs/                  ← 项目文档
│   ├── 需求文档.md         ← 功能需求详细说明
│   ├── 技术方案.md         ← 技术选型和架构
│   ├── 设计规范.md         ← UI 设计规范（含浅色/深色配色表）
│   └── 执行步骤.md         ← 开发步骤记录（已完成全部标记）
└── 开发日记/              ← 开发进展记录
    └── 2026-07-05.md
```

## 技术要点

- 单 HTML 文件，零外部依赖
- 数据存 localStorage，支持 JSON 导出/导入
- 纯白/纯黑两种主题，响应式设计
- SM-2 间隔重复算法
- 事件委托 + 静态模态框混合事件模式
- 可部署到 GitHub Pages

## 工作原则

1. **逐步推进**：每次只完成一小部分，完成后和用户沟通确认
2. **先解释再动手**：用大白话告诉用户要做什么、为什么
3. **用户视角**：用户不懂代码，沟通时描述功能而非实现
4. **文档同步**：改动后更新开发日记和 docs
