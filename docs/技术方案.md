# 技术方案

## 整体架构

**单 HTML 文件**（约 2200 行），包含 HTML、CSS、JavaScript，零外部依赖。

## 技术选型

| 考量 | 选择 | 原因 |
|------|------|------|
| 部署 | 单 HTML 文件 | 双击打开，可部署 GitHub Pages |
| 数据存储 | localStorage | 浏览器原生，无需服务器 |
| 跨设备 | JSON 复制/粘贴 + 文件导入导出 | 免费，通过微信即可传输 |
| 外观 | 纯 CSS + CSS 变量 | 无 UI 库依赖，支持浅色/深色/跟随系统 |
| 事件处理 | onclick + 静态模态框 addEventListener 混合 | 兼顾动态 HTML 和静态元素 |

## 数据结构

```javascript
{
  words: [{
    id: string,           // 唯一标识
    english: string,      // 英文
    chinese: string[],    // 中文释义列表
    createdAt: number,    // 创建时间戳
    stats: {
      totalAttempts, correctAttempts,  // 答题统计
      lastTested,                      // 上次测试时间
      consecutiveCorrect,              // 连续正确次数
      easeFactor, interval, nextReview // SM-2 参数
    }
  }],
  settings: {
    nickname,       // 昵称
    optionsCount,   // 每题选项数 (3-6)
    dailyCount,     // 每日挑战题数 (5-50)
    weakCount,      // 弱项突破题数 (5-50)
    darkMode,       // 'light' | 'dark' | 'auto'
    lastDailyDate,  // 上次完成每日挑战日期
    dailyStreak     // 连续打卡天数
  }
}
```

## SM-2 间隔重复算法

简化版 Anki 算法：
- 首次答对 → 10 分钟后重考
- 二次答对 → 1 小时后重考
- 之后 → 间隔 × 难度系数递增
- 答错 → 3 分钟后重考，难度系数降低
- 到期单词优先出现

## 页面结构（四标签页）

1. **测试**（首页）：问候语 + 三种模式选择 + 答题界面
2. **单词库**：添加/搜索/管理单词 + 批量操作
3. **统计**：全局数据 + 单词详情（排序/筛选）
4. **设置**：昵称/题数/外观/关于

## 关键设计模式

- **事件处理**：选项/提交按钮使用 onclick（动态 HTML），确认弹窗使用静态 addEventListener
- **DOM 驱动**：答题相关函数检查 DOM 状态而非 JS 变量，避免状态丢失
- **CSS 变量**：浅色/深色主题通过 CSS 变量切换，`prefers-color-scheme` 媒体查询支持跟随系统

## 浏览器兼容

支持所有现代浏览器（Chrome、Edge、Safari、Firefox）。
