# 加入词库指引

## 什么是云词库

云词库是预制的单词集合（如"高考基础词汇""四级进阶词汇"），存储在 Supabase 上。用户打开"拾词"→ 单词库 → 点"☁️ 云词库"即可浏览和导入。

你可以随时通过 Supabase SQL Editor 添加、修改、删除云词库，无需改代码。

---

## 数据库结构

```
word_banks 表：
├── id          自增编号
├── name        词库名（如"高考基础词汇"）
├── description 描述（如"高考英语大纲高频基础词汇"）
├── category    分类（初中 / 高中 / 四级 / 六级 / 考研 / 雅思 / 托福 等）
├── difficulty  难度（基础 / 进阶 / 拔高）
├── word_count  词数（和 JSON 里的实际数量一致）
└── words       JSONB 数组，每个词包含 english, chinese, pos
```

每个词的数据格式：
```json
{
  "english": "abandon",
  "chinese": ["放弃；抛弃"],
  "pos": "v."
}
```

| 字段 | 必须 | 说明 |
|------|------|------|
| english | ✅ | 英文单词 |
| chinese | ✅ | 中文释义，数组格式，每个元素是一条释义 |
| pos | 可选 | 词性（n. v. adj. adv. prep. conj. pron.） |

## 如何在 Supabase 后台操作

### 打开 SQL Editor

1. 打开 https://supabase.com → 登录 → 进入你的项目
2. 左侧菜单 → **SQL Editor** → 点 **New query**
3. 粘贴 SQL → 点右下角 **Run**

### 插入一个新词库

```sql
INSERT INTO word_banks (name, description, category, difficulty, word_count, words)
VALUES (
  '你要的词库名',
  '简短描述',
  '分类',
  '难度',
  单词数量,
  '[
    {"english":"apple","chinese":["苹果"],"pos":"n."},
    {"english":"banana","chinese":["香蕉"],"pos":"n."}
  ]'::jsonb
);
```

> ⚠️ 注意：`word_count` 要和 JSON 数组里的单词数一致，结尾的 `::jsonb` 不要漏掉。

### 用变量方式插入（更清晰）

如果词很多，建议用变量先定义 JSON：

```sql
DO $$
DECLARE
  v_words jsonb := '[
    {"english":"abandon","chinese":["放弃；抛弃"],"pos":"v."},
    {"english":"ability","chinese":["能力；才能"],"pos":"n."},
    {"english":"abnormal","chinese":["反常的；异常的"],"pos":"adj."}
  ]'::jsonb;
BEGIN
  INSERT INTO word_banks (name, description, category, difficulty, word_count, words)
  VALUES ('测试词库', '这是一个测试', '高中', '基础', jsonb_array_length(v_words), v_words);
END $$;
```

`jsonb_array_length(v_words)` 自动计算词数，不用手动数。

### 查看已有词库

```sql
SELECT id, name, category, difficulty, word_count, created_at
FROM word_banks
ORDER BY id;
```

### 预览某个词库的内容

```sql
SELECT name, words
FROM word_banks
WHERE id = 1;
```

### 修改词库

```sql
-- 改名字和描述
UPDATE word_banks
SET name = '新名字', description = '新描述'
WHERE id = 1;

-- 替换整个词库的单词
UPDATE word_banks
SET words = '[{"english":"new","chinese":["新的"],"pos":"adj."}]'::jsonb,
    word_count = 1
WHERE id = 1;
```

### 删除词库

```sql
DELETE FROM word_banks WHERE id = 1;
```

### 清空所有词库重新来

```sql
DELETE FROM word_banks;
-- 自增编号也会重置
ALTER SEQUENCE word_banks_id_seq RESTART WITH 1;
```

## 词性缩写对照

| 缩写 | 含义 |
|------|------|
| n. | 名词 |
| v. | 动词 |
| adj. | 形容词 |
| adv. | 副词 |
| prep. | 介词 |
| conj. | 连词 |
| pron. | 代词 |

如果有多个词性，用空格隔开：`"pos": "n. v."`

## 分类和难度建议

建议的分类（`category`）：

| 分类 | 适用人群 |
|------|---------|
| 初中 | 初中生 |
| 高中 | 高中生 |
| 四级 | 大学四级 |
| 六级 | 大学六级 |
| 考研 | 研究生考试 |
| 雅思 | 出国留学 |
| 托福 | 出国留学 |

建议的难度（`difficulty`）：

| 难度 | 含义 | 显示 |
|------|------|------|
| 基础 | ⭐ | 高频核心词 |
| 进阶 | ⭐⭐ | 中等难度 |
| 拔高 | ⭐⭐⭐ | 低频/难词 |

## JSON 格式注意事项

1. **必须用双引号**：JSON 键和字符串值只能用 `"`，不能用 `'` 或中文引号
2. **最后一项不要逗号**：`{"a":"b"},` 正确，`{"a":"b"},]` 错误
3. **中文释义用分号隔开**：`"chinese": ["苹果；水果"]` 多条释义用`；`
4. **特殊字符转义**：如果释义中有 `"`，写成 `\"`

### JSON 在线校验

如果不确定格式对不对，把 JSON 部分（`[{...},{...}]`）复制到 https://jsonlint.com 校验。

## 示例：完整插入高考基础词汇（前10个）

```sql
DO $$
DECLARE
  v_words jsonb := '[
    {"english":"abandon","chinese":["放弃；抛弃"],"pos":"v."},
    {"english":"ability","chinese":["能力；才能"],"pos":"n."},
    {"english":"abnormal","chinese":["反常的；异常的"],"pos":"adj."},
    {"english":"aboard","chinese":["在船/飞机上；上船/飞机"],"pos":"adv. prep."},
    {"english":"abroad","chinese":["在国外；到国外"],"pos":"adv."},
    {"english":"absence","chinese":["缺席；不在"],"pos":"n."},
    {"english":"absolute","chinese":["绝对的；完全的"],"pos":"adj."},
    {"english":"absorb","chinese":["吸收；吸引"],"pos":"v."},
    {"english":"abstract","chinese":["抽象的；摘要"],"pos":"adj. n."},
    {"english":"abundant","chinese":["丰富的；充裕的"],"pos":"adj."}
  ]'::jsonb;
BEGIN
  INSERT INTO word_banks (name, description, category, difficulty, word_count, words)
  VALUES ('高考基础词汇', '高考英语大纲高频基础词汇', '高中', '基础', jsonb_array_length(v_words), v_words);
END $$;
```
